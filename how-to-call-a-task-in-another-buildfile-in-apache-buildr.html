How to Call a Task in another Buildfile in Apache Buildr

by Tim Henderson
Fri 14 March 2014

asdf

Apache Buildr is a nice build system for Java and other JVM langauges. It is based on Ruby's Rake system and make it quite easy to setup multi-project builds. However, what if you want to re-use a Buildfile from a sub-project in a parent project? By default there is no real support for this. The intreprepid hacker can fix that problem by loading the sub-projects Buildfile like this:

.ruby::

if File::exists?('sub-project/Buildfile')
  pwd = Dir.pwd
  Dir.chdir File::join(pwd, 'sub-project')
  load "./Buildfile"
  puts project('sub-project').base_dir  ## HACK: pins the project to the
                                        ## proper directory!
  Dir.chdir pwd
end

If you don't access base_dir before you reset the current working director then it will eventually get set to the wrong directory causing build failures.

After you have loaded the Buildfile you can run:

$ pwd
/path/to/project
$ buildr help:projects
(in /path/to/project, development)
/path/to/project/sub-project
  main-project
  sub-project
  other-project
  another-project

All of the tasks from you sub-project should now work as expected from the parent project. Happy Buildring!
